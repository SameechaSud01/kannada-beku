Kannada Beku

What the app is and what it does — snapshot as of July 4, 2026 (branch fix/bug-fixes-and-audits).

What it is

Kannada Beku (formerly "Kannada Baa") is a mobile app that teaches conversational Kannada to non-Kannadiga adults living in Bengaluru — people who want to talk to auto drivers, shopkeepers, and in-laws, not pass an exam. The tone is deliberately warm, forgiving, and culturally proud, with a Karnataka green-and-gold identity throughout.

It ships for iOS and Android (tablet on iOS; no dark mode; web is dev-only). Built with Expo + Expo Router, TypeScript strict mode, Zustand (persisted via AsyncStorage) for client state, TanStack Query for server state, and Supabase for auth and progress storage. Audio is device text-to-speech (recorded native audio is post-MVP). Sentry error monitoring was wired in recently.

Main user journey

Sign up → short onboarding (name, motivation, daily commitment) → Lesson 0 "Basics" → work through 8 lessons → unlock and replay practice games per lesson part → track streaks, daily rings, and overall mastery.

Feature areas

1. Onboarding & Auth Live

  • Supabase email/password auth: login, forgot-password, reset-password (app/(auth)/).
  • Five-step onboarding: welcome → name → motivation → daily commitment → basics intro (app/onboarding/).

2. Lessons Live

  • Lesson 0 "Basics" — a listen-first, 7-step introductory flow (recently redesigned; content DB-sourced).
  • Lessons 1–8, content canonical in TypeScript (constants/lessons/lessonContent.ts), split into sub-sections: Greetings, Names, Wanting, Pointing, Easy verbs, Questions, Hard verbs, Putting it together.
  • Lesson flow follows scenario → intake → drill → output → done; unlocking is linear.
  • Content aligned to the owner's Kannada-Beku-Content.docx with ISO diacritic transliteration.

3. Practice games Live

  • Four game types defined in constants/games.ts: quiz, dictation, conversations, opposites.
  • Games unlock per completed lesson part and are replayable from the Practice tab (app/(games)/[game]/[n]).
  • Correct/wrong answers use green/red feedback (an approved exception to the warm-only palette).

4. Progress, streaks & mastery Live

  • Daily rings — real per-day Listen / Speak / Practice tracking, computed client-side.
  • Streak — counts learning days (not raw app-open days).
  • Overall mastery % — content-derived client-side rollup across all four games (services/progress/overallMastery.ts); the old server trigger is unused.
  • XP from game scores; offline actions sync through a queue store (stores/syncQueueStore.ts).

5. Reference content

  • Emergency phrase guide Live — offline phrase list in 3 categories, English-first, with TTS playback (app/emergency.tsx).
  • Beginner's Guide Live — script/alphabet reference, DB as source of truth with an offline TS fallback (app/guide/).
  • Heritage section Stub — cultural notes per topic; routing exists (app/heritage/[id]), content still TBD.

6. App shell & polish

  • Four tabs: Home (rings, streak), Learn (lesson path), Practice (games), Profile.
  • Settings: audio, feedback, help; local reminders (lib/reminders.ts).
  • System states built out: splash, loading, failure, empty (components/states/).
  • Playful "chunky" visual system: Baloo typeface, floating tab bar, scaled sizing everywhere via react-native-size-matters.
  • Production logging: centralized lib/logger.ts + Sentry (owner still owes the auth token / alert rule).

Explicit non-goals

  • No exam prep, no leaderboards or streak shaming, no dark mode, no other Indian languages, no recorded audio in v1.
